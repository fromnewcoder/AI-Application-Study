Week 5 Ph225 — Retrieval improvements: hybrid search, reranking, HyDE, MMR, and metadata filtering reference and demo

BM25

Sparse / keyword search

Term-frequency scoring (TF-IDF variant). Exact word matching with length normalisation. Fast, interpretable, no GPU.

+ Precise for rare technical terms
− Misses synonyms; no semantic understanding
Vector

Dense / semantic search

Embed query and docs into high-dimensional space. Cosine/dot-product similarity. Captures paraphrase and concepts.

+ Handles synonyms, intent, cross-lingual
− Misses exact rare terms; slow w/o ANN index
Hybrid = BM25 + Vector

Reciprocal Rank Fusion (RRF)

Merge two ranked lists without needing normalised scores. Each document gets score = Σ 1/(k + rank_i) across both lists (k=60 is standard). Best of both worlds — exact matches and semantic matches both surface.

Reranking

Cross-encoder

Pass (query, doc) pairs through a full transformer. Much more accurate than bi-encoder but slow — run on top-K only (e.g. top 50 → top 5).

HyDE

Hypothetical doc embedding

Generate a fake answer with an LLM, embed that. Query space shifts from question-space to answer-space, matching real docs better.

MMR

Maximal marginal relevance

Balance relevance vs diversity. Penalise docs too similar to already-selected ones. Prevents returning 5 near-duplicate chunks.

Metadata filtering

Pre-filter before retrieval

Attach structured metadata to chunks (date, source, author, topic). Filter at the vector DB level before similarity search — dramatically reduces noise without sacrificing speed. Always apply before reranking, not after.

The retrieval pipeline runs in stages. Each stage reduces the candidate set — retrieval casts wide, reranking narrows precisely.
1
Metadata pre-filter — query the vector DB with structured filters (date range, source, language). Candidate set: full corpus → filtered subset.
2
HyDE (optional) — generate hypothetical answer with LLM, embed it. Use this embedding as query instead of raw question embedding. Shifts query to answer-space.
3
BM25 retrieval — keyword search over filtered subset. Returns top-N ranked list (e.g. top 100). Fast.
4
Vector retrieval — ANN search over filtered subset. Returns top-N ranked list (e.g. top 100). Runs in parallel with BM25.
5
RRF fusion — merge both lists into a single ranking using 1/(k+rank). Candidates: union of both lists.
6
Cross-encoder reranking — score (query, doc) pairs together. Expensive but accurate. Takes top-50, returns top-5.
7
MMR selection (optional) — from reranked results, greedily pick docs balancing relevance and diversity. Final context: 3–5 diverse, high-relevance chunks.
Candidate sizes (typical)
Full corpus  100k docs
After metadata filter  ~5k
BM25 + vector each  100
After RRF  ~150
After cross-encoder  5–10
After MMR  3–5
When to use HyDE

Questions where the query phrasing differs a lot from how the answer is written. E.g. "how do I X" → docs contain "to X, you should…" HyDE bridges the linguistic gap.

When to skip HyDE

Low-latency apps (extra LLM call costs ~200ms), or when the query already sounds like a document (e.g. "transformer self-attention mechanism").

1 — BM25 (rank_bm25)
from rank_bm25 import BM25Okapi corpus = [doc.split() for doc in documents] bm25 = BM25Okapi(corpus) bm25_scores = bm25.get_scores(query.split()) bm25_ranked = sorted(enumerate(bm25_scores), key=lambda x: x[1], reverse=True)[:100]
2 — Vector search (sentence-transformers)
from sentence_transformers import SentenceTransformer import faiss, numpy as np model = SentenceTransformer("BAAI/bge-base-en-v1.5") doc_embs = model.encode(documents, normalize_embeddings=True) index = faiss.IndexFlatIP(doc_embs.shape[1]) index.add(doc_embs) q_emb = model.encode([query], normalize_embeddings=True) _, vec_ids = index.search(q_emb, 100) # top 100
3 — RRF fusion
def rrf_fuse(bm25_ids, vec_ids, k=60): scores = {} for rank, idx in enumerate(bm25_ids): scores[idx] = scores.get(idx,0) + 1/(k + rank + 1) for rank, idx in enumerate(vec_ids): scores[idx] = scores.get(idx,0) + 1/(k + rank + 1) return sorted(scores, key=scores.get, reverse=True) fused = rrf_fuse( [i for i,_ in bm25_ranked], vec_ids[0].tolist() )
4 — Cross-encoder reranking
from sentence_transformers import CrossEncoder reranker = CrossEncoder("cross-encoder/ms-marco-MiniLM-L-6-v2") pairs = [(query, documents[i]) for i in fused[:50]] ce_scores = reranker.predict(pairs) reranked = [fused[i] for i in np.argsort(ce_scores)[::-1][:10]]
5 — MMR selection
def mmr(query_emb, doc_embs, doc_ids, k=5, lam=0.5): selected, remaining = [], list(doc_ids) while len(selected) < k and remaining: rel = doc_embs[remaining] @ query_emb if selected: red = (doc_embs[remaining] @ doc_embs[selected].T).max(axis=1) else: red = np.zeros(len(remaining)) mmr_scores = lam * rel - (1 - lam) * red best = remaining[np.argmax(mmr_scores)] selected.append(best) remaining.remove(best) return selected
