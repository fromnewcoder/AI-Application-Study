Week 5 — Memory Patterns in LLM Chatbots: interactive reference and summarisation memory lab

1

In-context memory

Raw conversation history appended to every prompt. Simple, zero extra work.

+Exact recall, trivial to implement
−Hits token limit fast; cost grows linearly
2 ← Lab focus

Summarisation memory

After N turns, compress older messages into a rolling summary. Keep recent turns verbatim.

+Bounded token usage; long conversations
−Detail loss on summarised turns
3

Entity memory

Extract named entities (people, places, facts) and maintain a structured entity store.

+Precise recall of facts about specific entities
−Extraction errors; engineering complexity
4

External stores (Redis)

Persist memory outside the model — vector DB, Redis, or SQL. Retrieve relevant chunks at query time.

+Unlimited history; multi-session; multi-user
−Infrastructure overhead; retrieval quality

How summarisation memory works

Each turn

Append the new user/assistant pair to a recentTurns buffer.

After threshold (e.g. 6 turns)

Send the oldest half to the model with a "summarise this" prompt. Store the result as rollingSum. Drop those messages.

System prompt injection

Prepend rollingSum to every new call as part of the system message. The model sees: summary + recent turns.

Core logic — annotated

Three variables drive everything:

// State let recentTurns = []; // verbatim recent messages let rollingSum = ""; // compressed older context const THRESHOLD = 4; // summarise when recentTurns.length >= this const KEEP = 2; // keep this many recent turns after summarising
Sending a message
async function chat(userText) { recentTurns.push({ role:"user", content: userText }); // Build system message — inject summary if we have one const systemMsg = rollingSum ? `You are a helpful assistant.\n\nConversation so far (summary):\n${rollingSum}` : "You are a helpful assistant."; const response = await callClaude(systemMsg, recentTurns); recentTurns.push({ role:"assistant", content: response }); // Trigger summarisation if threshold reached if (recentTurns.length >= THRESHOLD) await summarise(); return response; }
Summarisation step
async function summarise() { // Split: compress everything except the last KEEP pairs const toCompress = recentTurns.slice(0, -KEEP); recentTurns = recentTurns.slice(-KEEP); const prevContext = rollingSum ? `Previous summary:\n${rollingSum}\n\n` : ""; const prompt = prevContext + "New conversation segment:\n" + toCompress.map(m => `${m.role}: ${m.content}`).join("\n") + "\n\nWrite a concise summary that preserves key facts, decisions, and names."; rollingSum = await callClaude("You are a summariser.", [ { role:"user", content: prompt } ]); }
Key insight

The rolling summary is cumulative — when you summarise again, you pass the old summary + the new segment. This prevents info loss across multiple compression rounds.
