Core concepts
01
Tool definitions
Declare each tool with a name, description, and typed parameter schema. The model selects tools based on this spec.
02
Tool chaining
Output of one tool becomes input for the next. The agent decides the chain at runtime based on task decomposition.
03
Error handling
Tool failures must return structured errors back to the model so it can retry, reroute, or gracefully abort.
04
Stopping conditions
Define when the agent should stop: task complete, max turns hit, ambiguity detected, or user confirmation needed.

Key implementation concepts
Parallel vs sequential tool calls
When tool calls are independent, dispatch them in parallel to cut latency. When output A feeds tool B, they must be sequential. The model signals this implicitly through argument structure.
tool_result with is_error flag
Always return a tool_result even on failure. Set is_error: true and include a message. The model will see this and decide whether to retry, use a fallback tool, or explain the failure to the user.
Context window management
Tool results accumulate in the context. Long file contents or search dumps can blow the window. Truncate or summarize results before returning them — never pass raw multi-page text back verbatim.
Stop reason: end_turn vs tool_use
Check the API response's stop_reason. end_turn means the model is done. tool_use means it needs another round. Loop on tool_use, exit on end_turn.
Agent loop skeleton
Python · anthropic SDK
def run_agent(user_msg, tools, max_turns=10):
    messages = [{"role":"user","content":user_msg}]

    for turn in range(max_turns):
        resp = client.messages.create(
            model="claude-sonnet-4-20250514",
            tools=tools,
            messages=messages
        )

        # stopping conditions ──────────────────────
        if resp.stop_reason == "end_turn": break
        if turn == max_turns - 1: raise MaxTurnsError

        # tool dispatch ────────────────────────────
        tool_results = []
        for block in resp.content:
            if block.type == "tool_use":
                result = dispatch(block.name, block.input)
                tool_results.append({
                    "type": "tool_result",
                    "tool_use_id": block.id,
                    "content": result.content,
                    "is_error": result.is_error,
                })

        # append assistant turn + results ──────────
        messages += [
            {"role":"assistant","content":resp.content},
            {"role":"user","content":tool_results},
        ]

    return resp
Stopping conditions
✓ Complete
Task satisfied
Model returns end_turn with a final text response. Normal, happy exit.
✗ Max turns
Turn budget exceeded
Hard cap on loop iterations. Raise an error or return partial result to avoid runaway agents.
⚠ Tool error
Unrecoverable failure
Multiple retries failed. Agent surfaces the error to the user rather than looping infinitely.
? Ambiguous
Needs clarification
Model asks the user a question instead of using a tool. Pause the loop, collect input, resume.
