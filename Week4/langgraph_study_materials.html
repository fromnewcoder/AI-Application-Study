What is LangGraph?
LangGraph is a library built on top of LangChain for building stateful, multi-step agentic applications as explicit graphs. Instead of sequential chains, you model your logic as nodes connected by edges — giving you fine-grained control over branching, looping, and persistence.
State
A TypedDict (or Pydantic model) that acts as shared memory across all nodes. Every node reads from it and returns a partial update — only the keys it changed.
Nodes
Plain Python functions (or async functions / runnables). Each node receives the full state and returns a dict of updated keys. Side-effectful work happens here: LLM calls, tool use, I/O.
Edges
Directed connections between nodes. A normal edge always goes A → B. A conditional edge routes to different nodes depending on a router function's return value.
Compile + invoke
graph.compile() validates the graph and returns a CompiledGraph. Call .invoke(state) to run it synchronously, or .stream() for step-by-step updates.
Conditional routing
add_conditional_edges(source, path_fn, path_map) registers a router. The path_fn inspects the current state and returns a string key; the path_map maps that key to the target node name. This is how you build branching logic without if/else scattered through your nodes.
Graph visualisation
LangGraph can export a Mermaid diagram of your graph structure. After compiling, call app.get_graph().draw_mermaid() for the Mermaid source, or .draw_mermaid_png() to get PNG bytes directly. Great for debugging and documentation.

1 — Define state
State is the single source of truth. All nodes receive it and return partial updates.
from typing import TypedDict class ChatState(TypedDict): user_input: str # raw user message intent: str # "math" | "weather" | "unknown" response: str # final reply
2 — Write nodes
Each node is a function: state in → partial dict out.
def detect_intent(state: ChatState) -> dict: text = state["user_input"].lower() intent = "math" if "add" in text else "unknown" return {"intent": intent} # only update intent key def handle_math(state: ChatState) -> dict: return {"response": f"Solving: {state['user_input']}"}
3 — Build + add conditional edges
Register nodes, set the entry point, wire up conditional routing.
from langgraph.graph import StateGraph, END graph = StateGraph(ChatState) graph.add_node("detect_intent", detect_intent) graph.add_node("handle_math", handle_math) graph.set_entry_point("detect_intent") # router function returns next node name def route(state): return f"handle_{state['intent']}" graph.add_conditional_edges( "detect_intent", route, {"handle_math": "handle_math", ...} ) graph.add_edge("handle_math", END)
4 — Compile, visualise, invoke
Compile validates structure; invoke runs the graph to completion.
app = graph.compile() # visualise png = app.get_graph().draw_mermaid_png() with open("graph.png", "wb") as f: f.write(png) # invoke result = app.invoke({ "user_input": "add 3 + 4", "intent": "", "response": "" }) print(result["response"])
LangGraph quick reference
StateGraph(State)
Create a new graph. Pass your TypedDict as the schema — LangGraph enforces key types at runtime.
add_node(name, fn)
Register a node. The name is a string used in edge definitions. fn receives state and returns a partial dict.
set_entry_point(name)
Mark which node runs first. Equivalent to adding an edge from the implicit START node.
add_edge(src, dst)
Unconditional edge. After src completes, always go to dst. Use END as dst to terminate.
add_conditional_edges
Takes (source, path_fn, path_map). path_fn(state) → key; path_map maps key → node name. Enables branching.
graph.compile()
Validates the graph (checks for dangling nodes, missing edges) and returns a runnable CompiledGraph.
app.invoke(state)
Run the graph synchronously from entry point to END. Returns the final merged state dict.
app.stream(state)
Generator that yields {node_name: partial_state} after each node runs. Useful for debugging.
get_graph().draw_mermaid()
Returns Mermaid diagram source as a string. Paste at mermaid.live to render.
draw_mermaid_png()
Returns PNG bytes of the rendered graph. Requires Playwright (pip install playwright).
END
Sentinel imported from langgraph.graph. Adding an edge to END signals graph completion from that node.
