Tool / Function Calling Basics
Tool schema definition · how the model decides to call tools · parsing tool_use responses · tool result format
01 · Schema
Tool definition
A JSON object with name, description, and an input_schema (JSON Schema). The description is the most important part — the model reads it to decide when and how to call the tool.
02 · Decision
When the model calls
The model reads user intent, compares it against each tool's description, and emits a tool_use block when a tool is needed. It can call multiple tools in one turn.
03 · Parsing
tool_use response block
The response content array contains blocks of type: "tool_use" with an id, name, and input object. You must check for this type before calling the real API.
04 · Result
tool_result format
You call the real tool, then send back a message with role: "user" containing a tool_result block referencing the tool_use_id so the model can generate a final answer.

Key insight: The model never actually calls your API. It emits a structured request (tool_use), you run the real code, then feed the result back. The model is the orchestrator — your code is the executor.

1
User sends a message
"What's the weather in Singapore?" — natural language, no awareness of tools required.
2
You send the request with tools array
Your app sends the conversation + the tool schema to /v1/messages. The model reads both.
3
Model returns stop_reason: "tool_use"
Response contains a tool_use block with id, name: "get_weather", and input: {location: "Singapore"}.
4
Your code calls the real weather API
You inspect the tool_use block, extract input.location, and make the actual HTTP call to the weather service.
5
Send tool_result back to the model
Append the assistant's tool_use turn to history, then add a new user turn with type: "tool_result" containing the weather data.
6
Model generates final answer
The model reads the tool result and produces a natural-language response: "It's 31°C and humid in Singapore right now."
// Tool schema — the model reads the description to decide when to call const tools = [ { name: "get_weather", description: "Get current weather for a city. Call this whenever the user asks about weather, temperature, or forecast.", input_schema: { type: "object", properties: { location: { type: "string", description: "City name, e.g. 'Singapore' or 'Paris, France'" }, unit: { type: "string", enum: ["celsius", "fahrenheit"], description: "Temperature unit. Default celsius." } }, required: ["location"] } } ];
// Send the request with tools attached const response = await fetch("https://api.anthropic.com/v1/messages", { method: "POST", headers: { "Content-Type": "application/json" }, body: JSON.stringify({ model: "claude-sonnet-4-20250514", max_tokens: 1000, tools: tools, // ← attach your tools array messages: [ { role: "user", content: "What's the weather in Singapore?" } ] }) }); const data = await response.json(); // data.stop_reason will be "tool_use" — not "end_turn"
// Parse the tool_use block from the response const toolUseBlock = data.content.find( block => block.type === "tool_use" ); if (toolUseBlock) { const { id, name, input } = toolUseBlock; // id → "toolu_01XVcD..." (use this when sending result) // name → "get_weather" // input → { location: "Singapore", unit: "celsius" } // Dispatch to the right handler if (name === "get_weather") { const weather = await callWeatherAPI(input.location, input.unit); // → { temp: 31, condition: "Humid", humidity: 84 } } }
// Send tool result back — role must be "user" const finalResponse = await fetch("https://api.anthropic.com/v1/messages", { method: "POST", headers: { "Content-Type": "application/json" }, body: JSON.stringify({ model: "claude-sonnet-4-20250514", max_tokens: 1000, tools: tools, messages: [ { role: "user", content: "What's the weather in Singapore?" }, { role: "assistant", content: data.content }, // full assistant turn { role: "user", content: [{ type: "tool_result", tool_use_id: toolUseBlock.id, // must match! content: JSON.stringify(weather) // your API result }] } ] }) }); // Model now produces the final natural-language answer
