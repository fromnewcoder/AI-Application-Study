What is the ReAct Pattern?
ReAct = Reason + Act. It's a prompting paradigm where an LLM interleaves thinking steps with tool-use actions, creating a grounded, auditable reasoning loop. Introduced by Yao et al. (2022).

↺ loop
Reason
Think step
The agent generates a thought — what it knows, what's missing, what tool would help. This is the internal scratchpad before acting. Makes reasoning transparent and debuggable.
Act
Action step
A structured call to an external tool: web search, calculator, database query, API. The action must be deterministic and parseable. Usually formatted as tool_name(args).
Observe
Observation step
The tool result injected back into context. This grounds the agent in real-world data, preventing hallucination and enabling multi-hop reasoning over live information.
Why ReAct beats chain-of-thought alone Chain-of-thought reasons from memory only — it can hallucinate facts. ReAct grounds each reasoning step in observed tool outputs, dramatically improving accuracy on tasks requiring up-to-date or precise information.
The Prompt Template
Each iteration is formatted into the LLM's context window as a structured dialogue. Tools and their descriptions are listed in the system prompt.
// System prompt (once at start) You are a helpful agent. You have access to these tools: - search(query: str) → web results - calc(expression: str) → numeric result - lookup(entity: str) → factual info Format each step EXACTLY as: Thought: [your reasoning] Action: tool_name(arg) Observation: [tool result — filled by system] ... repeat until ... Final Answer: [your answer] // User turn What is the population of Tokyo × the GDP per capita of Japan? // Agent loop adds steps incrementally: Thought: I need Tokyo population first. Action: search("Tokyo population 2024") Observation: Tokyo population ≈ 13.96 million Thought: Now I need Japan's GDP per capita. Action: lookup("Japan GDP per capita 2024") Observation: $33,834 USD Thought: I can calculate this now. Action: calc("13960000 * 33834") Observation: 472,522,640,000 Final Answer: ~$472.5 billion
Key insight: the observation closes the loop Without injecting the observation back into context, the next Thought step would be blind to what happened. The context window IS the agent's working memory.
Implementing the ReAct Loop
The core loop is simple. A while loop calls the LLM, parses the response, routes to the correct tool, injects the result, and checks for a stop condition.
import re, anthropic client = anthropic.Anthropic() def react_agent(question: str, tools: dict, max_iter: int = 10) -> str: """Run a ReAct loop until Final Answer or max iterations.""" messages = [ {"role": "user", "content": question} ] for iteration in range(max_iter): # ← MAX ITERATIONS GUARD # 1. REASON — call the LLM response = client.messages.create( model="claude-opus-4-5", system=SYSTEM_PROMPT, messages=messages, max_tokens=512 ) text = response.content[0].text # 2. Check for Final Answer (STOP CONDITION) if "Final Answer:" in text: return text.split("Final Answer:")[1].strip() # 3. Parse the Action action_match = re.search( r"Action:\s*(\w+)\((.+?)\)", text ) if not action_match: break # malformed — stop gracefully tool_name, tool_arg = action_match.groups() # 4. ACT — call the tool if tool_name not in tools: observation = f"Error: unknown tool '{tool_name}'" else: observation = tools[tool_name](tool_arg.strip('"')) # 5. OBSERVE — inject result into context messages.append({"role": "assistant", "content": text}) messages.append({ "role": "user", "content": f"Observation: {observation}" }) # MAX ITERATIONS REACHED — return partial answer return "Reached max iterations. Partial result from last observation."
Max Iterations Guard
Without a hard cap on iterations, an agent can loop forever (infinite loops on bad tool outputs, circular reasoning). The guard is non-negotiable.
✓ Natural stop

Agent outputs Final Answer:. The task is solved. Clean exit — return the answer to the caller.

⚠ Max iterations

Loop counter hits max_iter. Return best partial answer and log a warning. This is a safety net, not an expected path.

✗ Tool error

Tool returns an error. Agent should reason about the error in the next Thought and try a different approach. After N consecutive errors — stop.

⟳ Repetition detect

Agent issues the same action twice in a row. This signals a stuck loop. Detect and break early with a "stuck" message.

What max_iter should you use? Simple lookups: 3–5 iterations. Complex multi-hop research: 10–15. Never set it above 20 unless you have very long, well-defined tasks. Tokens cost money and time — budget your iterations.
# Enhanced guard with repetition detection prev_action = None consecutive_errors = 0 for i in range(max_iter): # ... (get response, parse action) ... # Repetition guard current_action = f"{tool_name}({tool_arg})" if current_action == prev_action: return "Agent stuck in repetition. Aborting." prev_action = current_action # Error streak guard if "Error" in observation: consecutive_errors += 1 if consecutive_errors >= 3: return "Too many consecutive tool errors." else: consecutive_errors = 0 # reset on success

3 Multi-Step Problems — Worked Solutions
Each problem demonstrates a different ReAct pattern. Study the thought traces — they reveal the agent's decision-making process.
PROBLEM 1 Multi-source fact aggregation
"Who is the current CEO of OpenAI, and what was their role before?"
Iteration 1 Thought: I need to find the current OpenAI CEO. Action: search("OpenAI CEO 2024") Observation: Sam Altman is CEO of OpenAI as of 2024. Iteration 2 Thought: I have the name. Now I need their prior role. Action: lookup("Sam Altman career history") Observation: Sam Altman was president of Y Combinator (2014–2019) before joining OpenAI. Iteration 3 Thought: I have both pieces of info. I can answer now. Final Answer: Sam Altman is the current CEO of OpenAI. Before this role, he served as president of Y Combinator from 2014 to 2019.
Pattern: Serial lookups — each observation narrows the question for the next action. 3 iterations, clean stop.
PROBLEM 2 Math + lookup composition
"If I invest $10,000 at the current US Fed funds rate for 2 years (compounded annually), what do I get back?"
Iteration 1 Thought: I need the current Fed funds rate — I don't know this from memory. Action: search("US Federal funds rate 2024") Observation: The Fed funds rate is 5.25%–5.50% as of late 2024. I'll use 5.375% (midpoint). Iteration 2 Thought: Now I apply compound interest formula: A = P(1+r)^t Action: calc("10000 * (1 + 0.05375) ** 2") Observation: 11103.89 Iteration 3 Thought: I have the result. The gain is $1103.89. Ready to answer. Final Answer: At a 5.375% annual rate, $10,000 grows to $11,103.89 after 2 years — a gain of $1,103.89.
Pattern: Lookup → Compute — real data fetched first, then calculation. Never assume the rate from training data. This prevents stale-knowledge errors.
PROBLEM 3 Error recovery + branching
"Find the birthplace of the director of the 2023 film Oppenheimer, then find a famous landmark there."
Iteration 1 Thought: First I need to find who directed Oppenheimer (2023). Action: search("Oppenheimer 2023 film director") Observation: Christopher Nolan directed Oppenheimer (2023). Iteration 2 Thought: Now I need Nolan's birthplace. Action: lookup("Christopher Nolan birthplace") Observation: Error: entity not found in knowledge base. Iteration 3 — ERROR RECOVERY Thought: The lookup tool failed. I'll try search instead for the same info. Action: search("Christopher Nolan born where") Observation: Christopher Nolan was born in Westminster, London, England. Iteration 4 Thought: Birthplace is London. Now find a famous landmark there. search("famous landmark Westminster London") Observation: The Palace of Westminster (Houses of Parliament) is in Westminster, London. Iteration 5 Thought: I have everything needed to answer fully. Final Answer: Oppenheimer (2023) was directed by Christopher Nolan, who was born in Westminster, London. A famous landmark there is the Palace of Westminster.
Pattern: Error recovery — when a tool fails, Reason suggests an alternative tool. This is key to robust agents. Notice how the agent didn't give up — it adapted.
