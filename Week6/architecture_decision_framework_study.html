Four architecture patterns

Pattern 01

RAG

Retrieval-Augmented Generation. Fetch relevant context at runtime and inject into the prompt.

Dynamic knowledge No retraining Citeable
Pattern 02

Fine-tune

Adapt a base model's weights on domain-specific examples to shift its behavior or style.

Style/format control No runtime cost Costly upfront
Pattern 03

Agent

LLM as an orchestrator. Calls tools, searches, writes code, loops until task complete.

Multi-step tasks Tool use High complexity
Pattern 04

Simple prompt

Zero-shot or few-shot prompts. No pipelines, no training. Just the right system prompt.

Fast iteration Low cost Limited scope
