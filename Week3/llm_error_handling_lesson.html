Error taxonomy

CodeNameRetryable?Cause
429 Rate limit yes TPM/RPM quota hit. Respect retry-after header
500 Server error yes Transient failure on Anthropic side. Safe to retry
529 Overloaded yes API capacity. Back off and retry
400 Bad request no Malformed payload. Fix the input, don't retry blindly
401 Unauthorized no Invalid API key. Human intervention needed
— Malformed JSON maybe Model didn't follow format. Re-prompt or parse defensively
— Timeout / network yes Transient connectivity. Retry with backoff

Core strategies

⏱
Exponential backoff
Wait grows by 2ⁿ between retries. Prevents thundering herd and hammering the API. Add jitter to desync clients.
🔁
Max retry budget
Cap total attempts (e.g. 3–5). After budget is exhausted, throw a final error or activate a fallback path.
🔍
Output validation
Parse and schema-check every response. If malformed, re-prompt with corrective instructions before giving up.
🪂
Fallback strategy
When retries fail: degrade gracefully. Return cached result, use smaller model, queue for async, or surface an informative error.

Exponential backoff with jitter

Base (s) 1.0 Multiplier 2.0 Max (s) 60

Orange bars = pure exponential. Blue dots = with jitter (±30%). Formula: min(base × mult^attempt + rand(), maxWait)

Robust wrapper — async Python

import asyncio, random, json, time from typing import Any import anthropic client = anthropic.Anthropic() class LLMError(Exception): def __init__(self, msg, retryable=False): super().__init__(msg) self.retryable = retryable RETRYABLE_CODES = {429, 500, 502, 503, 529} async def call_with_retry( messages: list, model: str = "claude-sonnet-4-20250514", max_retries: int = 4, base_delay: float = 1.0, max_delay: float = 60.0, schema: dict | None = None, ) -> Any: """Call Anthropic API with exponential backoff + output validation.""" for attempt in range(max_retries + 1): try: resp = client.messages.create( model=model, max_tokens=1024, messages=messages, ) content = resp.content[0].text # ── Output validation ──────────────────────── if schema: parsed = _parse_json(content) # raises on malformed _validate(parsed, schema) # raises on schema mismatch return parsed return content except anthropic.APIStatusError as e: retryable = e.status_code in RETRYABLE_CODES if not retryable or attempt == max_retries: raise LLMError(f"API {e.status_code}: {e.message}", retryable=retryable) wait = _backoff(attempt, base_delay, max_delay) print(f"[retry {attempt+1}] status={e.status_code} wait={wait:.1f}s") await asyncio.sleep(wait) except (ValueError, KeyError) as e: # Malformed / schema-invalid output if attempt == max_retries: raise LLMError(f"Output invalid after {max_retries} retries") # Inject corrective instruction messages.append({"role": "assistant", "content": content}) messages.append({"role": "user", "content": f"Your last response was invalid JSON or missing required fields. " f"Error: {e}. Please respond with valid JSON only."}) await asyncio.sleep(0.5) def _backoff(attempt: int, base: float, cap: float) -> float: "Exponential backoff with full jitter." exp = min(base * (2 ** attempt), cap) return random.uniform(0, exp) # full jitter strategy def _parse_json(text: str) -> dict: "Strip markdown fences, parse JSON." clean = text.strip().strip("```json").strip("```").strip() return json.loads(clean) # raises ValueError on bad JSON def _validate(data: dict, schema: dict) -> None: for key, typ in schema.items(): if key not in data: raise KeyError(f"Missing field: {key}") if not isinstance(data[key], typ): raise ValueError(f"{key} expected {typ.__name__}")

Fallback wrapper

1
Model downgrade
If Sonnet fails, retry with Haiku. Cheaper, faster, often enough for the task.
2
Cached response
Return a recent successful response for identical/similar inputs. Useful for read-heavy pipelines.
3
Async queue
Enqueue the request to a job queue (e.g. Celery, BullMQ). Process later when rate limit resets.
4
Graceful degradation
Return a deterministic rule-based result or a helpful "service unavailable" message. Never a silent failure.
